## 三、虚拟机侧（做一次准备 + 一条日常命令）

### 环境前提（你已具备，无需重装）

```bash
export JAVA_HOME=/usr/local/jdk1.8.0_162
export HADOOP_HOME=/usr/local/hadoop
export SPARK_HOME=/usr/local/spark
export PATH=$JAVA_HOME/bin:$SPARK_HOME/bin:$HADOOP_HOME/bin:$PATH

start-dfs.sh && start-yarn.sh   # 若未启动
```

### 一次性准备（**只跑这一次**）

```bash
cd /media/sf_cs-/charging-bigdata
bash scripts/vm-setup-once.sh
```

自动完成：CRLF 修复 → JDBC 下载 → MySQL 连通测试 → HDFS 上传四文件。

### 全量流水线（日常 / 答辩前）

```bash
cd /media/sf_cs-/charging-bigdata
bash scripts/run-pipeline.sh
```

顺序固定：**MR v1~v7 → MySQL MR 表 → Spark ADS → MySQL ADS 表**

### 验收

```bash
bash scripts/verify-pipeline.sh
```

期望：

- MR 七张表有数据
- `t_charging_daily` 起始于 `20141118`（nvv2t 会话，`created` 规范化后）
- `t_soc_hourly` 时间正序：`2019072611` → `2019072614` → `2019072700`（dsv13r2）

### 分段重跑（MR 已成功时）

```bash
# 只重跑 Spark ADS + MR 入库
SKIP_MR=1 bash scripts/run-pipeline.sh

# 只 Spark ADS
bash spark/scripts/run-spark-ads.sh
```

## 五、常见问题（不用再改虚拟机）

| 现象 | 处理 |
|------|------|
| `$'\r': 未找到命令` | `sed -i 's/\r$//' config/pipeline.env` 后重跑；新版脚本会自动处理 |
| `MYSQL_PASSWORD 为空` | 编辑 `config/pipeline.env` 填写密码 |
| `HDFS 输入不存在` | `bash scripts/vm-setup-once.sh` |
| `缺少 MySQL JDBC` | `vm-setup-once.sh` 会自动 wget |
| 充电图显示 2014 起 | 正常：nvv2t 会话数据为 2014–2015；与 dsv13r2 的 2019 是不同数据源 |
| 共享目录脚本无执行权限 | 用 `bash scripts/xxx.sh`，不要 `chmod +x` |
