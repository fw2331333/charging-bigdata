# 三节点 Hadoop 集群部署手册

> 目标：真三节点（1 Master + 2 Slave），副本数 3，SSH 免密，与 `scripts/run-pipeline.sh` 无缝对接。  
> 架构背景见 [企业级架构与ETL设计.md](./企业级架构与ETL设计.md)。

---

## 0. 节点规划

| 主机名 | 建议 IP | 角色 |
|--------|---------|------|
| hadoop-master | 192.168.1.101 | NN、RM、DN、NM |
| hadoop-slave1 | 192.168.1.102 | DN、NM |
| hadoop-slave2 | 192.168.1.103 | DN、NM |

三台统一：

- OS：CentOS 7.x / Ubuntu 18.04+（与课程一致可用 CentOS 7）
- JDK 8
- Hadoop **3.1.3**（与 `mapreduce/pom.xml` 一致）
- 用户 `hadoop`，`HADOOP_HOME=/opt/hadoop-3.1.3`

---

## 1. 主机名与 hosts

三台 `/etc/hosts` 内容一致，参考：

```bash
cat hadoop/cluster/cluster-hosts.example
# 按实验室 IP 修改后写入三台 /etc/hosts
```

Master 上：

```bash
sudo hostnamectl set-hostname hadoop-master
```

Slave 同理设置为 `hadoop-slave1` / `hadoop-slave2`。

---

## 2. 安装 Hadoop（每台或仅 Master 后 rsync）

**方式 A — Master 先装，再同步目录（推荐）**

在 Master 上以 root 执行（仅首次）：

```bash
cd ~/charging-bigdata/hadoop
sudo bash install-pseudo-distributed.sh
```

再将 `/opt/hadoop-3.1.3` 与 `/etc/profile.d/hadoop.sh` 同步到两台 Slave（路径一致）。

**方式 B — 每台单独执行安装脚本**

确保 `HADOOP_HOME`、数据目录 `/data/hadoop` 三台完全相同。

---

## 3. SSH 免密互访

在 **hadoop-master** 上以 `hadoop` 用户执行：

```bash
bash ~/charging-bigdata/hadoop/cluster/setup-ssh-cluster.sh
```

验证：

```bash
ssh hadoop-slave1 hostname
ssh hadoop-slave2 hostname
```

> `start-dfs.sh` / `start-yarn.sh` 会通过 `workers` 文件 SSH 到各节点启动进程，免密是硬性要求。

---

## 4. 部署集群配置

在 **hadoop-master** 上以 `hadoop` 用户：

```bash
export HADOOP_HOME=/opt/hadoop-3.1.3
source /etc/profile.d/hadoop.sh
bash ~/charging-bigdata/hadoop/cluster/deploy-cluster-config.sh
```

脚本会：

1. 将 `hadoop/conf-cluster/` 下 **core / hdfs / yarn / mapred / workers / hadoop-env** 写入 `$HADOOP_HOME/etc/hadoop/`
2. 在三台创建 `/data/hadoop/{tmp,hdfs/namenode,hdfs/datanode}`
3. rsync 配置到 slave1、slave2

### 从单机扩为三节点时注意

- **若单机已有 HDFS 数据且不能丢**：不要执行 `hdfs namenode -format`
- 扩集群后执行 `hdfs dfs -setrep -R 3 /Car` 可提升已有文件副本数

---

## 5. 启动集群

仅在 **hadoop-master**：

```bash
source /etc/profile.d/hadoop.sh
start-dfs.sh
start-yarn.sh
jps
```

Master 上应看到：`NameNode`、`ResourceManager`，以及本机 `DataNode`、`NodeManager`。

---

## 6. 验证三节点

```bash
bash ~/charging-bigdata/hadoop/cluster/verify-cluster.sh
```

期望：

- `hdfs dfsadmin -report` → **3** 个 Live datanodes
- `yarn node -list` → **3** 个 RUNNING NodeManager

Web UI：

- HDFS：`http://hadoop-master:9870`
- YARN：`http://hadoop-master:8088`

## 7. 上传数据与跑流水线

```bash
# 一次性准备（若未做过）
bash scripts/vm-setup-once.sh

# 全量离线流水线：MR → ETL → Spark ADS → MySQL
bash scripts/run-pipeline.sh
```

MySQL 仍在 Windows 宿主机（或独立 DB 服务器），虚拟机通过 `10.0.2.2` 或 `config/pipeline.env` 中的 `MYSQL_HOST` 连接。

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 只有 1 个 DataNode | 检查 `workers` 是否三台；Slave 上 `HADOOP_HOME` 是否一致；SSH 免密 |
| `start-dfs.sh` 卡住要密码 | 重新运行 `setup-ssh-cluster.sh` |
| 副本数仍为 1 | 确认已用 `conf-cluster/hdfs-site.xml`；对新文件自动为 3 |
| MR 报 Connection refused 9000 | Master 上 `start-dfs.sh`；`fs.defaultFS` 主机名可解析 |
| ETL 连不上 MySQL | Windows 防火墙、`bind-address`、`sql/allow_root_remote.sql` |
