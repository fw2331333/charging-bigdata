# 离线批处理流水线



> **虚拟机操作请看唯一权威文档：[虚拟机一次性运行手册.md](./虚拟机一次性运行手册.md)**  

> 下文仅作架构说明，不再单独维护多套命令。



## 架构



```text

bash scripts/run-pipeline.sh

  ├─ 预检：Hadoop / HDFS / MySQL

  ├─ MapReduce JobRunner（dsv13r1 → v1~v7 → HDFS /Car/output）

  ├─ load_mr_to_mysql.py（MR → Windows MySQL）

  └─ Spark SQL ADS（dsv13r2 + nvv2t → MySQL：电池分析 + 充电行为）

              ↓

    FastAPI + ECharts（只读 MySQL）

```



## 快速命令



```bash

# 首次（虚拟机）

cd /media/sf_cs-/charging-bigdata

bash scripts/vm-setup-once.sh



# 日常全量

bash scripts/run-pipeline.sh



# 验收

bash scripts/verify-pipeline.sh

```



## 配置项（config/pipeline.env）



| 变量 | 含义 |

|------|------|

| `HDFS_INPUT` | MR 输入，固定 `/Car/dsv13r1.csv` |

| `HDFS_ADS_DSV` | ADS 电池分析，`/Car/dsv13r2.csv` |
| `HDFS_ADS_NVV` | ADS 充电行为，`/Car/nvv2t.csv` |

| `DATASET_LOCAL_DIR` | 本地上传 CSV 目录 |

| `MYSQL_HOST` | Windows 宿主机 `10.0.2.2` |

| `MYSQL_JDBC_JAR` | Spark 写 MySQL 用驱动 |

| `SKIP_MR` / `SKIP_ETL` / `SKIP_ADS` | 分段调试 |



ADS 使用 **dsv13r2 + nvv2t**（指导书 §3.2 / §3.3），不用 nvv2t 做 ML（ML 用 nvv2t_md §3.4）。
